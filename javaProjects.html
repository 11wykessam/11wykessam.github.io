Random Generation Word Art

This was my first every attempt at a graphical project in java. If I were to do it again I would structure the project almost entirely differenty. For a start confining everything to one text file made the code tricky to read and almost impossible to trace errors. It would have been infinitely more suitable to have individual classes for different aspects of the program e.g the menu, canvas and circles. However I am proud of what this project stands for: it represents a shift away from small maths programs in a console to something I can show to people. I remember distinctly being extremely proud when I finished this. The art generated is also very special to me. In the program each letter has a unique shape within the program, with each circle being of random size and colour. This means that whilst the same string input will return the same general shape. The colours and sizes of the circles will be different each time. All the images sampled on this site have been generated from words that are very special to me - names of family, my hobbies and secrets. It represents a way for me to share my life publicly without actually telling anyone any specifics about me. From then, I like to think I've grown as a programmer and looking back on this early project made me happy and nostalgic.

Modular Arithmetic Circle Art

This is one of my earliest graphical java projects. Before I learnt JavaFX and before I even knew how to properly write object orientated code (this is mainly characterised by 14 year old me's bizarre decision to put almost all of the code into one text file). Nevertheless the principles that this project stood for still remain very dear to me. The idea of beauty behind the most mundane mathematics and finding complex patterns within simple bits of maths. This project works on the principle that every number is connected by a line to itself times the number inputted into the times table box then modulo the number in the divisions box. This simple bit of maths can create some extraordinary, from cardioids set to squares. I was heavily inspired for this project by the youtube channel Mathologer. Towers Of Hanoi

I decided to do this project to practice animation. Primarily trying to use a queue to implement animation. Each sprite has a queue of versions of itself, it will change to the next sprite every frame at a rate of 60 per second. This was also a good exercise in object orientation as I was able to create subclasses for the pins and have all things on the screen as being part of the superclass Sprite. Then to animate I could just implement a tick and render method in Sprite and iterate through the entire Sprite list ticking and rendering 60 times a second per sprite. Overall I think this was quite succesful and the code I have created will form the basis for a new project I am making.

Logic Circuit Simulator

This is my most ambitious project so far and has been my pet project for the last few months. It is an amalgamation of everything that I have learned so far from programming. The premise is a simple circuit diagram drawer and simulator similar to Yenka however it is more simplified and deals only with logic diagrams. I found the limitations of Yenka's small canvas quite annoying so my solution has an infinite canvas, the way in which you can draw wires is also much less limited than Yenka with the option of snapping and horizontal/vertical wires. Current features include drawing, simulating and saving circuits and I'm planning on adding an eraser tool for easier removal of components and more components in future. I've learned loads from the development of this project including saving objects more easily, more about polymorphism and how to manage more complicated data structures. All skills I look forward to developing in future.
